Create Remote Development Server

Introduction

There are circumstances when you local machine is not suitable for heavy processing or its operating system is incompatible with your target environment. While it is possible to find a workaround or a compromise, this article examines an alternative solution.

There is nothing new in having data stored in one system and having it processed in another. This article will provide a pragmatic approach towards creating a remote development server where source code modification will happen locally and any processing will be done remotely.

The setup will consist of the following:

  • DigitalOcean
  • Ubuntu 20.04 LTS (droplet image)
  • OpenSSH (preinstalled on image)
  • Node and nginx (development tooling)
  • rsync and inotifywait

Create Droplet

Sign-up on DigitalOcean and create droplet. When choosing between a password-based authentication and SSH, I choose password based and configure SSH keys/config myself later (also explained here). As mentioned in the beginning, I choose Ubuntu distro version 20.04 LTS (why I haven't chosen Docker is explained below). Choose Droplet specs according to your workload needs.
